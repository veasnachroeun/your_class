HTML Charset

⇝ A charset, ឬ charater set គឺជាសំណុំនៃកូដតួអក្សរ ដែលទទួលស្គាល់ដោយ html
⇝ សម្រាប់ HTML5 ស្តង់ដា charset ដែលប្រើ គឺ UTF-8


ដើម្បីអោយ html ផេក បង្ហាញចេញអោយបានត្រឹមត្រូវ, ចាំបាច់ Browser ត្រូវតែស្គាល់ រាល់ charater set ដែលបានប្រើលើ html ខាងក្រោមជាឧទាហរណ៌ នៃការប្រើប្រាស់ charater set លើ html


ជាឧទាហរណ៏៖

<meta charset=UTF-8>

⇝ UTF-8 មិនត្រូវបានយកមកប្រើគ្រប់ជំនាន់នោះទេ។ ជាក់ស្តែងសម្រាប់ WEB ជំនាន់មុន គឺប្រើ ASCII.

⇝ ចាប់ពីជំនាន់ HTML 2.0 ដល់ HTML 4.01, ISO-8859-1 ត្រូវបានចាត់ទុក ជាស្តង់ដា charater set.

⇝ ជាមួយនឹង XML និង​ HTML5, UTF-8 ជាជំនាន់ចុងក្រោយ ហើយវាបានជួយដោះស្រាយរាល់បញ្ហាជាច្រើន លើ charater encoding.


In the beginning: ASCII



⇝ ទិន្នន័យកុំព្យូទ័រត្រូវបានរក្សាទុកជាលេខកូដគោលពីរ (01000101) នៅក្នុងអេឡិចត្រូនិក។

⇝ ដើម្បីធ្វើការក្សាទុក អត្ថបទជាលក្ខណះស្តង់ដា, The American Standard Code for Infomation Interchange (ASCII) ត្រូវបានបង្កើតឡើង។ វាត្រូវបានកំណត់ជាលេខកូដគោលពីរ ដែលមានតែមួយគត់អាចរក្សាទុកបានរាល់តួអក្សរ​និងលេខ និមួយៗ ចាប់ពីលេខ 0​ ដល់​ 9 អក្សរធំនិងអក្សរតូច ពី (a ដល់ z, A ដល់ Z) និងតួអក្សរពិសេសៗ មួយចំនួនទៀតដូចជា !​ $ + - () @ < > , .

⇝ តាំងពី ASCII ប្រើបាន 7 bits សម្រាប់តំណាងតួអក្សរទាំងអស់ គឺប្រើបានតែ 128 តួរ ប៉ុណ្ណោះ លើអក្សរនិងលេខខុសៗគ្នា

⇝ ភាពកម្សោយ ធំបំផុតរបស់ ASCII គឺថាវាមិនរួមបញ្ចូល តួអក្សរផ្សេង ក្រៅពី អង់គ្លេស

⇝ ASCII គឺ នៅតែប្រើប្រាស់រហូតដល់សព្វថ្ងៃ ជាពិសេស នៅប្រពន្ឋ័ដំណើរការកុំព្យួទ័រធំៗ

⇝ សម្រាប់ការសិក្សាលំអិត មើលមេរៀន ASCII ពេញ

In Windows: Windows-1252



⇝ Windows-1252 ជាតួអក្សរលំនាំដើមក្នុង Windows, ចាប់តាំងពី windows 95​។

⇝ វាជាផ្នែកបន្ថែមមួយទៅកាន់ ASCII, រួមបញ្ចូលជាមួយតួអក្សរជាអន្តរជាតិ។

⇝ វាប្រើជា full byte(8-bite)ដែលមានតួអក្សរតំណាង 256​ តួខុសគ្នា

⇝ ចាប់តាំងពី Windows-1252 ក្លាយជាលំនាំដើមក្នុង Windows វាត្រូវបានទទួលស្គាល់ពី browser ទាំងអស់

⇝ សម្រាប់ការសិក្សាលំអិត មើលមេរៀន Windows-1252 ពេញ

In HTML 4: ISO-8859-1



⇝ សំណុំតួអក្សរដែលប្រើញឹកញា៉ប់ និង ច្រើនបំផុតក្នុង HTML 4 គឺ ISO-8859-1
ISO-8859-1 ជាការបន្ថែមទៅកាន់ ASCII, ជាមួយនឹងការដាក់បព្ចូលបន្ថែមតួអក្សរជាអន្តរជាតិ


ជាឧទាហរណ៏៖

<meta http-equiv="Content-Type" Content="text/html;charset=ISO-8859-1">

⇝ នៅក្នុង HTML 4, សំណុំតួអក្សរដែលខុសគ្នា ពី ISO-8859-1 អាចកំណត់បានដូចក្នុង <meta> tag ខាងក្រោម៖


ជាឧទាហរណ៏៖

<meta http-equiv="Content-Type" Content="text/html;charset=ISO-8859-8">

⇝ គ្រប់ HTML 4 processors ទាំងអស់ ក៏ support ជាមួយ UTF-8 ផងដែរ


ជាឧទាហរណ៏៖

<meta http-equiv="Content-Type" Content="text/html;charset=UTF-8">

ពេលដែល browser កំណត់បាន ISO-8859-1, ជាធម្មតាវាជាលំនាំដើម Windows-1252, ព្រោះ Windows-1252 មាន 32 តួអក្សរអន្តរជាតិ បន្ថែម


⇝ សម្រាប់ការសិក្សាលំអិត មើលមេរៀន ISO-8859-1 ពេញ



In HTML5: Unicode UTF-8



⇝ លើការប្រើប្រាស់ HTML5 កំណត់អោយអ្នកបង្កើតវេបសាយ ប្រើ UTF-8 ជាសំណុំតួអក្សរ


ជាឧទាហរណ៏៖

<meta charset="UTF-8">

⇝ សំណុំតួអក្សរដែលខុសគ្នា ពី UTF-8 អាចកំណត់បានដូចក្នុង tag ខាងក្រោម៖


ជាឧទាហរណ៏៖

<meta charset="ISO-8859-1">

⇝ The UTF-8 និង UTF-16, ជាប្រពន្ឋ័ Unicode ស្តង់ដាដែលបានបង្កើតឡើង, The ISO-8859 ជាសំណុំតួអក្សរមានកំណត់, ហើយវាក៏មិនស្របគ្នានិងភាសារផ្សេងនោះទេ។

⇝ The Unicode Standard គ្របដណ្តប់ ស្ទើរគ្រប់តួអក្សរ​ និង និមិត្តសញ្ញាទាំងអស់នៅលើពិភពលោក។


គ្រប់ HTML5 និង XML processors support UTF-8, UTF-16, Windows-1252, និង​ ISO-8859.


⇝ សម្រាប់ការសិក្សាលំអិត មើលមេរៀន Unicode UTF-8 ពេញ
