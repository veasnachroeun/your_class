ជម្រើសប្រើតួអក្សរតូចចំពោះ (Element Names)

ក្នុងការសរសេរកូដលើHTML អាចអនុញ្ញាតិអោយធ្វើការសរសេរ អក្សរធំ លាយជាមួយអក្សរតូចបាន, តែទោះជាយ៉ាងនេះក្តី ជម្រើដែលប្រសើរបំផុតសម្រាប់អ្នកគឺត្រូវប្រើអក្សរតូច ជាចម្រើសដែលល្អ៖

  • ការប្រើអក្សរធំលាយជាមួយអក្សរ​តូច មើលទៅមិនល្អ
  • សំរាប់អ្នកជំនាញវេបសាយ ប្រើអក្សរតូចជាគោលក្នុងការសរសេរ
  • អក្សរតូចមានរបៀបរៀបរយ
  • អក្សរតូច ងាយសរសេរ

ជម្រើសប្រើតួអក្សរតូចចំពោះ (Attribute Names)

ក្នុងការសរសេរកូដលើHTML អាចអនុញ្ញាតិអោយធ្វើការសរសេរ អក្សរធំ លាយជាមួយអក្សរតូចបាន ទៅលើ Attribute Names, តែទោះជាយ៉ាងនេះក្តី ជម្រើដែលប្រសើរបំផុតសម្រាប់អ្នកគឺត្រូវប្រើអក្សរតូច ជាចម្រើសដែលល្អ៖

  • ការប្រើអក្សរធំលាយជាមួយអក្សរ​តូច មើលទៅមិនល្អ
  • សំរាប់អ្នកជំនាញវេបសាយ ប្រើអក្សរតូចជាគោលក្នុងការសរសេរ
  • អក្សរតូចមានរបៀបរៀបរយ
  • អក្សរតូច ងាយសរសេរ

ជម្រើសប្រើសញ្ញាឆ្មេញកណ្តុរ (" ") ចំពោះ (Attribute Value)

ក្នុងHTML យើងមិនចាំបាច់ប្រើ សញ្ញា ឆ្មេញកណ្តុរ (" ") ចំពោះAttribute Names ក៏បាន តែទោះជាយ៉ាងនេះក្តី ជម្រើដែលប្រសើរបំផុតសម្រាប់អ្នកគឺត្រូវប្រើវាជាប្រចាំ ជាចម្រើសដែលល្អ៖

  • សំរាប់អ្នកជំនាញវេបសាយ ប្រើសញ្ញាឆ្មេញកណ្តុរជាប្រចាំ
  • ប្រើសញ្ញាឆ្មេញកណ្តុរ ងាយអាន
  • យើងត្រូវប្រើសញ្ញា ធ្មេញកណ្តុរ ប្រសិនបើតម្លៃនៅខាងក្នុង មាន ការដកឃ្លា

<title> Element មិនគួររម្លង

   <title> element ជាតម្រូវការចាំបាច់ក្នុង HTML

ការដាក់ចំណងជើងលើផេក <title> ពិតជាសំខាន់ សម្រាប់បង្កើនប្រសិទ្ឋិភាពដល់ម៉ាសុីនស្វែងរក (SEO), ប្រើ Page title ដើម្បីផ្តល់ទៅអោយម៉ាសុីន ធ្វើការ រាយបញ្ចីឈ្មោះផេក តាមលំដាប់ ទៅតាមការរុករក

the <title> Element:

  • កំណត់ចំណងជើងនៅលើ Browser toolbar
  • ផ្តល់ជាចំណង់ជើងសម្គាល់ផេក ពេលដែល ដាក់ចូល មុខងារ facorites
  • បង្ហាញចេញចំណងជើង ផេក នៅលើ ម៉ាសុីនស្វែងរក

ហេតុដូច្នេះ, ត្រូវដាក់ចំណងជើង ផេកអោយបានត្រឹមត្រូវ នឹងមានន័យគ្រប់គ្រាន់តាមអាចធ្វើបាន

កាលដែលមិនប្រើ <html> tag និង <body> tag

   នៅលើ កូដ html វាក៏អាចដំណើរការបានដូចគ្នា ដោយមិនចាំបាច់បញ្ចូល <html> និង <body>tage ទៅលើការសរសេរកូដ។


for example

<!DOCTYPE html>
<head>
<title>Your Class all in one</title>
</head>

<h1>This is heading</h1>
<p>This is paragraph</p>


   តែទោះជាកូដដំណើរការបានដោយគ្មា <html> និង <body>tage យ៉ាងណាក្តី, សម្រាប់អ្នកជំនាញ អភិវឌ្ឍន៏ វេបសាយ វាចាំបាច់ជាទីបំផុត ក្នុងការបញ្ចូល <html> និង <body> ចូលទៅលើការសរសេរកូដ។

  • ការលុប <body> ចេញអាចមិនដំណើរការលើ browser ចាស់ៗ
  • ការលុប​ <html> និង <body> អាចធ្វើអោយគាំងនៅលើកម្មវិធី DOM និង XML


Good Coding

<!DOCTYPE html>
<html>
<head>
<title>Your Class all in one</title>
</head>
<body>
<h1>This is heading</h1>
<p>This is paragraph</p>
</body>
</html>

កាលដែលប្រើ lang attribute

   ជានិច្ចជាកាល គួរប្រើ lang attribute នៅខាងក្នុង <html> tag, ដើម្បីប្រកាសចេញជាភាសារ លើវេបផេករបស់យើង, វាមានន័យថា​ ជាជំនួយមួយផ្នែក ផ្តល់ទៅកាន់ ម៉ាសុីនស្វែងរកលើ browser.


Good Coding

<!DOCTYPE html>
<html lang ="en-us">
<head>
<title>Your Class all in one</title>
</head>
<body>
<h1>This is heading</h1>
<p>This is paragraph</p>
</body>
</html>



កាលដែលប្រើ Meta Data

   គឺដើម្បីអោយប្រាកដថា ការបកស្រាយ ក៏ដូចជាការបញ្ជាក់ចេញពី ម៉ាសុីរុករក គឺត្រឹមត្រូវនិងសមស្រប ទាំំងភាសារ ហើយនិងតួអក្សរ <meta charset="charset"> វាគួរត្រូវបានកំណត់អោយបានមុខគេ នៅលើ html document.


Good Coding

<!DOCTYPE html>
<html lang ="en-us">
<head>
<meta charset ="UTF-8">
<title>Your Class all in one</title>
</head>
<body>
<h1>This is heading</h1>
<p>This is paragraph</p>
</body>
</html>



កាលដែលប្រើ Viewport

   ដើម្បីអាចអោយវេប ផេក មានការផ្លាស់ប្តូរ ប្រែប្រួល រីករួមទៅតាមទំហំ លើការប្រើឧបករណ៏ ផ្សេងៗគ្នា វានឹងរួមតូច នៅពេលដែលយើងបើកផេកនៅលើ អេក្រង់ទូរស័ព្ទ, រីកធំនៅពេលបើកនៅលើអេក្រង់កុំព្យូទ័រ,

ហេតុដូច្នេះ យើងគួរតែប្រើវានៅលើ <meta> element គ្រប់ វេបផេកទាំងអស់ដោយប្រើកូដដូចខាងក្រោម៖


Viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0">


Good Coding

<!DOCTYPE html>
<html lang ="en-us">
<head>
<meta charset ="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Your Class all in one</title>
</head>
<body>
<h1>This is heading</h1>
<p>This is paragraph</p>
</body>
</html>


នេះគឺជាការនែនាំលើ Browser លើរបៀបក្នុងការគ្រប់គ្រង ទំហំក៏ដូចជាការរីករួមរបស់ផេក

The width=device-width ជាការកំណត់យកផ្នែកទទឹងលើ ផេក អោយត្រូវទៅតាមទំហំអេក្រង់ ទទឹងលើឧបករណ៏ (វានឹងប្រែប្រួលទៅតាមទំហំរបស់ឧបករណ៏)

The initial-scale=1.0 ជាផ្នែកមួយដែលកំណត់ ការរីករួម កម្រិតដំបូង នៅពេលចាប់ផ្តើមដំណើតលើ Browser

ការប្រើ តួអក្សរលើឈ្មោះ file មួយៗ

   មាន web servers មួយចំនួន (Apache, Unic) គឺប្រកាន់យកតួអក្សរធំ ឬតូច នៅពេលដែលយើងដាក់ ឈ្មោះសម្គាល់ នៅលើ file, e.g "picture.jpg" មិនជៀ្រតចូលបានទេ បើយើសរសេរបែបនេះ "Picture.jpg"។

   web servers ផ្សេងទៀត​ដូចជា(Microsoft, IIS) គឺមិនប្រកាន់តួអក្សរនោះទេ យើងអាចកំណត់ឈ្មោះ file ជាអក្សរតូចឬធំបាន, e.g "picture.jpg" អាចបានប្រសិនបើយើងសរសេរបែបនេះ "PictuRe.jpg"

   ប្រសិនបើអ្នកសរសេរលាយអក្សរតូចនិងអក្សរធំចូលគ្នា អ្នកត្រូវកំណត់ចំណាំគ្រប់ចំណុចទាំងអស់នេះ, នៅពេលដែលអ្នកធ្វើការផ្លាស់ប្តូរ ពី server មួយដែលមិនប្រកាន់តួអក្សរ ទៅកាន់ server មួយផ្សេងទៀងដែលប្រកាន់យកតួអក្សរ នោះនឹងធ្វើអោយមានការ Errors ទៅលើដំណើរការ Web site ដោយសារមូលហេតុតូចៗទាំងនេះ។

ចំណាំ៖ ដើម្បីជៀសវៀងពីបញ្ហាអស់ទាំងនេះ គ្រប់ការប្រើប្រាស់ឈ្មោះ file ទាំងអស់ត្រូវ ប្រើជាអក្សរតូចជានិច្ច។

ការប្រើប្រាស់ Defaulf Filenames

   Defaulf filenames ជាការកំណត់សម្គាល់ផេកដំបូងដើម្បីបង្ហាញចេញ នៅលើ browser រាល់ពេលអ្នកប្រើប្រាស់ ចុចចូលមកកាន់វេបសាយ​ ដូចជា ("https://www.domainname.com"), sever និង ធ្វើការ ទាញយក default filename, ដូចជា​ "index.htm", "index.html" ឬ "default.htm" ជា តំណភ្ជាប់ដាក់លើ domainnames

   ក្នុងករណី sever របស់យើង ចាប់យកតែឈ្មោះ index.html ជា default filename យើងត្រូវដាក់ឈ្មោះ ផេកដំបូងរបស់យើង ជា "index.html" មិនមែនជា "default.html" នោះទេ

   ផ្ទុយមកវិញ បើសិនជា server អាចចាប់យក default filename ច្រើនជាងមួយ, នោះយើងអាច កំណត់ឈ្មោះផេករបស់យើងបានតាមតែចង់បាន។
